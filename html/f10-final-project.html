CS810 -- Advanced Programming in the UNIX Environment - Fall 2010 - Final Project

Final Project: Write a simple web server

Summary

The objective of this assignment is for you to write a simple web server that speaks a limited version of HTTP/1.0 as defined in RFC1945. This gives you occasion to understand the process of reading an RFC and implementing a well-known protocol. Furthermore, your implementation will be in two parts: the simple main eventloop, already implemented in HW#3 and a shared library. This gives you opportunity to understand the difference between writing your own program and implementing an API.

At the end, your HW#3 submission should be able to be linked against any other student's final project library and vice versa.

Problem assignment

Download all files in this directory. Replace the file main.c with your HW#3 submission, then implement and extend the libsws.so library as needed. You may add additional files to build the library or add code in the existing files, but you may not extend the file sws.h (as it defines the API all students follow).

As you fill in the library, you will implement the sws(1) command as described in the manual page provided to you here.

As usual, the general homework guidelines apply.

This is a non-trivial programming assignment -- do not postpone starting to work on this! Try to first get the basics right, then add functionality one by one, always ensuring that the added code did not break any previous functionality. If you're stuck, describe your problem on the mailing list.

Additional considerations

Please carefully read RFC1945 to understand version 1.0 of the Hyper Text Transfer Protocol.

Please carefully read http://en.wikipedia.org/wiki/Common_Gateway_Interface to understand the Common Gateway Interface.

Your program should behave as one would expect from a regular system daemon. That is, it should detach from the controlling terminal and run in the background, accept multiple simultaneous connections, not generate any messages on stdout unless in debugging mode etc.

Your program does have to return normal HTTP/1.0 status codes as described in the RFC.

Your program does NOT need to support IPv6.

Your program does NOT need to support the POST method.

Your program does NOT need to support the "-s dir -k key" options.

Your program does NOT need to support logging.

Extra Credit

The following are ways to gain extra-credit. Please only start working on these features if you are satisfied with the basic functionality.

  • Implementing support for IPv6: +10
  • Implementing support for logging: +10
  • Implementing support for POST: +30
  • Implementing support for encrypted content: +50
