CS810 -- Advanced Programming in the UNIX Environment - Fall 2010 - HW#1

HW#1: a simple test

Summary

The objective of this assignment is for you to get used to writing test cases prior to writing any programs. The test you write for this assignment will then be used lateron to verify the program you will write in the next assignment.

Problem assignment

Write a test to verify that a given program correctly copies one file to anther. You may write your test in any language you may wish, though a simple shell script might be fully sufficient in this case. Some of the things to consider:

  • verify the program correctly handles missing or wrong input arguments
  • verify the program correctly handles missing or wrong input files (source or target is a directory, lacking read- or write access, etc.)
  • verify the program correctly handles relative and absolute paths
  • verify the program returns the correct return code
  • verify the source and target are identical after the program has finished

Deliverables and Due Date

You will submit a single tar archive named username-hw1.tar that will extract into a directory named after your username. In that directory will be at least a Makefile and whatever other files you created to implement the test. If you have any commentary on your submission, include it in a file callead README. The Makefile has a single 'all' target that runs your test. The name of the program to be tested is given via the PROG=name variable.

For example, I would submit the file jschauma-hw1.tar which might extract the tests and let me run them as follows: 

$ tar xvf jschauma-hw1.tar
jschauma
jschauma/hw1
jschauma/hw1/test-cp
jschauma/Makefile
$ cd jschauma
$ make PROG=/bin/cp
$
